# 概要
ステラソラの物資画面のスクリーンショットから素材数を解析し、
各素材の所持数を表示する。

# 開発環境
- Windows 11
- Docker

# 使い方
1. ステラソラの物資画面のスクリーンショットを撮る
2. https://nabedroid.github.io/stellasora-inventory-ocr/ にアクセス
3. 「画像をドラッグ＆ドロップ・・・」をクリックする
4. ステラソラの物資画面のスクリーンショットを選択する（複数枚選択可）
5. 「解析開始」ボタンをクリックする
6. 解析結果が表示される
7. 画面上部の「サマリー表示に切替」をクリックする

## 設定
OCR の認識精度が低い場合、画面上部の「設定」からパラメータを調整可能です。

### ORB 最小一致点数
デフォルトは 5 です。
読み取ったアイコン特徴量とマスターデータのアイコン特徴量を比較し、
一致点数がこの値以上の場合、アイコンを認識したと判断します。

### 早期リターン閾値
デフォルトは 10 です。
一致点数がこの値以上の場合、それ以降の比較を打ち切ります。
本来は読み取ったアイコンとマスターデータの全てのアイコン特徴量を比較し、
最も一致点数が高いものを採用しますが、
一致点数がこの値以上の場合、それ以降の比較を打ち切ることで、
処理速度を向上させることができます。
処理速度が気にならない場合は値を 100 にしてください。

### 色許容誤差
デフォルトは 30 です。
読み取ったアイコンとマスタデータのアイコンの色相の差分がこの値以下の場合、
アイコンの色が一致したと判断します。
カセットなど色違いのアイテムが区別できない場合は値を小さくしてください。


## ローカルでの実行

1. Docker がインストールされた Windows の ターミナル（powershell）で以下のコマンドを実行

```
# ビルド及び開発サーバーでアプリが起動します
# ホットリロードが必要なら --watch を付けて起動
docker compose up main
# マスターデータを自作する場合
docker compose up manager
```

2. ブラウザで http://localhost:3000/ にアクセス（manager を起動した場合は 3001）
